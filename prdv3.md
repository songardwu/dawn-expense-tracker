# 出帳管理系統 V3｜產品需求文件 PRD

**版本：** v3.0
**日期：** 2026-04-15
**基於：** V2 已上線版本（認證 + 資料隔離 + 帳號管理）
**分支：** v3-dev

---

## 1. 版本目標

V3 聚焦「財務安全與管理效率」，在 V2 的認證基礎上新增：

1. **發票號碼防呆** — 阻擋重複發票，避免重複請款
2. **管理員全欄位 inline 編輯** — 一頁式核對工作台，不用進子頁面
3. **案場鎖定機制** — 管理員鎖定整個案場，所有提報進入唯讀
4. **廠商相似性即時比對** — 打字時即時提醒相似廠商，減少重複建檔
5. **基礎審計軌跡** — 記錄每筆修改的操作人與操作時間

---

## 2. 名詞定義

| 名詞 | 說明 |
|------|------|
| 案場 | 即「案場名稱」欄位（project_no），代表一個施工專案 |
| 鎖定 | 管理員對某案場執行鎖定後，該案場下所有提報不可編輯、不可刪除 |
| 相似廠商 | 廠商名稱中包含相同關鍵字（如「公司」「行」「工作室」等）的既有廠商 |

## 3. 資料庫異動

### 3.1 reports 表新增欄位

```sql
ALTER TABLE reports ADD COLUMN is_locked BOOLEAN NOT NULL DEFAULT FALSE;
ALTER TABLE reports ADD COLUMN updated_by INTEGER REFERENCES users(id);
ALTER TABLE reports ADD COLUMN updated_at TIMESTAMP;
```

- `is_locked`：由管理員按案場批次設定，鎖定後該筆不可編輯/刪除
- `updated_by`：最後修改人的 user_id（首次建立時為 NULL）
- `updated_at`：最後修改時間（首次建立時為 NULL）

### 3.2 vendor_keywords 表（新建）

```sql
CREATE TABLE vendor_keywords (
    id SERIAL PRIMARY KEY,
    keyword VARCHAR(50) NOT NULL UNIQUE
);
```

MVP 階段由工程師在 DB 直接寫入關鍵字，例如：

```sql
INSERT INTO vendor_keywords (keyword) VALUES
('公司'), ('行'), ('工作室'), ('企業'), ('設計'), ('工程');
```

> v1.1 再做管理員 UI 介面維護關鍵字。

## 4. 功能規格

### 4.1 發票號碼重複防呆

**觸發時機：** 設計師或管理員送出提報時（POST /submit）

**邏輯：**
1. 若 `invoice_no` 為空（選填），跳過檢查
2. 若 `invoice_no` 不為空，查詢 `SELECT id FROM reports WHERE invoice_no = %s`
3. 若已存在，回傳錯誤訊息「此發票號碼已存在，請確認是否重複請款」，表單保留已填內容
4. 檢查範圍：全系統（不分使用者），因為同一張發票不應出現在任何人的提報中

**管理員編輯時：** 修改發票號碼也要做重複檢查（排除自己那筆）

### 4.2 管理員全欄位 inline 編輯

**適用角色：** 僅管理員

**可編輯欄位（全部）：**

| 欄位 | 輸入方式 |
|------|---------|
| 名稱（vendor） | text input |
| 款項分類（category） | 下拉選單：案場成本 / 管銷 / 獎金 |
| 請款金額（amount） | number input |
| 發票收據編號（invoice_no） | text input |
| 發票收據日期（invoice_date） | date input |
| 匯款日期（remit_date） | date input |
| 案場名稱（project_no） | text input |

**操作方式：**
- 桌面版：表格每個儲存格直接是輸入框，每列一個「儲存」按鈕
- 手機版：卡片內每個值旁邊有小鉛筆 icon，點擊切換為輸入框，有「儲存」按鈕
- 儲存時打 POST，成功後刷新頁面

**鎖定狀態下：** 該案場的提報所有欄位顯示為純文字（不可編輯），「儲存」和「刪除」按鈕消失

**設計師：** 清單頁維持現狀（純檢視），不提供 inline 編輯

### 4.3 案場鎖定機制

**適用角色：** 僅管理員

**鎖定操作：**
- 清單頁新增「案場管理」區塊或按鈕
- 顯示目前所有案場名稱清單，每個案場旁有「鎖定 / 解鎖」切換按鈕
- 點擊「鎖定」→ `UPDATE reports SET is_locked = TRUE WHERE project_no = %s`
- 點擊「解鎖」→ `UPDATE reports SET is_locked = FALSE WHERE project_no = %s`

**鎖定效果：**
- 鎖定只影響已有的資料，不阻擋新增提報（設計師仍可對已鎖定案場提報新款項）
- 被鎖定的提報在清單頁顯示鎖頭 icon 🔒
- 管理員：編輯欄位變為唯讀文字、「儲存」「刪除」按鈕消失
- 設計師：原本就不能編輯，額外顯示「已鎖定」標記
- 新提報的 is_locked 預設為 FALSE，即使該案場有其他已鎖定的提報

**Excel 匯出：** 鎖定狀態不影響匯出，所有資料正常匯出

### 4.4 廠商相似性即時比對

**觸發時機：** 設計師或管理員在提報表單的「名稱」欄位打字時

**前端行為：**
1. 使用者在「名稱」欄位輸入文字
2. 輸入停頓 300ms 後（debounce），前端發 GET 請求到 `/api/check-vendor?q=使用者輸入`
3. 後端回傳相似廠商清單（JSON）
4. 前端在輸入框下方顯示提醒：「系統中已有相似廠商：XX公司、YY公司。是否為同一廠商？」
5. 提醒為**警告性質**，不阻擋送出

**後端比對邏輯（MVP 關鍵字版）：**
1. 從 `vendor_keywords` 取出所有關鍵字
2. 從使用者輸入中移除所有關鍵字，得到「核心名稱」
3. 查詢 `SELECT DISTINCT vendor FROM reports WHERE vendor != 使用者輸入`
4. 對每個既有廠商也移除關鍵字得到「核心名稱」
5. 若核心名稱相同或使用者輸入包含既有廠商的核心名稱，視為相似
6. 回傳相似廠商清單

**範例：**
- 使用者輸入「大明工程公司」
- 關鍵字清單：公司、工程
- 核心名稱：「大明」
- 資料庫中有「大明設計公司」→ 核心名稱也是「大明」→ 命中，提醒

> v1.1 升級為 80% 文字重疊率演算法。

### 4.5 基礎審計軌跡

**記錄時機：** 管理員透過 inline 編輯修改任何欄位時

**記錄內容：**
- `updated_by`：修改人的 user_id
- `updated_at`：修改時的時間戳記

**顯示方式：**
- 桌面版表格：新增「最後修改」欄，顯示「修改人 · 時間」
- 手機版卡片：卡片底部顯示「最後修改：修改人 · 時間」
- 若 `updated_by` 為 NULL，顯示「—」

> v1.1 升級為記錄舊值 vs 新值的詳細對照表。

## 5. 刻意不做的事（V3 範圍外）

| 不做的東西 | 原因 |
|-----------|------|

| 審計軌跡舊值/新值對照 | v1.1，MVP 只記修改人+時間 |
| 關鍵字管理 UI | v1.1，MVP 由工程師 DB 寫入 |
| 設計師自行修改密碼 | v1.1 |
| 批次匯入提報 | 不做 |
| 通知系統（email / LINE） | 不做 |
| 附件上傳（發票照片） | 不做 |

## 7. API 新增

| 路由 | 方法 | 說明 |
|------|------|------|
| `/api/check-vendor` | GET | 廠商相似性即時比對，參數 `q`，回傳 JSON |
| `/update-report/<id>` | POST | 管理員全欄位更新（含發票防呆 + 鎖定檢查 + 審計寫入） |
| `/toggle-lock-project` | POST | 管理員鎖定/解鎖案場，參數 `project_no` |

## 8. MVP 驗收標準

V3 完成後，必須通過以下測試：

### 權限與隔離
1. 設計師 A 登入，看不到設計師 B 的任何資料
2. 設計師無法存取管理員的編輯功能與案場管理

### 發票防呆
3. 設計師提報填入已存在的發票號碼，送出被阻擋，顯示紅字警告
4. 管理員編輯時改成已存在的發票號碼（排除自己），被阻擋
5. 發票號碼為空時不做重複檢查（選填欄位）

### 案場鎖定
6. 管理員鎖定案場 X 後，案場 X 的所有提報的編輯按鈕消失
7. 管理員鎖定案場 X 後，設計師仍可對案場 X 新增提報（鎖定只影響已有資料）
8. 管理員解鎖案場 X 後，編輯功能恢復
9. 被鎖定的提報在清單顯示鎖頭標記

### 管理員編輯
10. 管理員可 inline 修改所有欄位（名稱、金額、分類、案場、發票日期、發票號碼、匯款日期）
11. 修改後 `updated_by` 和 `updated_at` 正確寫入
12. 清單頁顯示最後修改人與時間

### 廠商相似性比對
13. 提報表單輸入「大明工程公司」，資料庫有「大明設計公司」，顯示相似提醒
14. 提醒為警告性質，使用者仍可送出
15. 輸入完全不相似的名稱，不顯示提醒

### 報表
16. Excel 匯出的金額加總與網頁清單頁完全一致
17. 鎖定狀態不影響 Excel 匯出
